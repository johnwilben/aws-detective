CASE #002 - THE CRIME SCENE

What Happened:

TechCorp stored 10,000 customer photos on AWS. This morning, they couldn't find them! The photos are critical evidence for their business.

🔍 CLUE #1: Click to Examine

Witness Statement:

"We uploaded the photos to something called a 'bucket'. It was supposed to store unlimited files..."

Detective's Note: A "bucket" that stores unlimited files? This narrows down our suspects!

🔍 CLUE #2: Click to Examine

Technical Evidence:

File types: .jpg, .png, .mp4 (photos and videos)

Total size: 500 GB

Detective's Note: Large files... We need a service designed for file storage!

Based on the clues, we're looking for an AWS service that:
  • Stores files (photos, videos)
  • Uses "buckets"
  • Can handle large amounts of data
CASE #003 - THE SUSPECTS

We have 5 suspects. Each is an AWS service. Click on each to learn about them:

📦
SUSPECT #1: Amazon S3

Alias: "Simple Storage Service"

Known For: Storing files in "buckets"

Specialty: Can store unlimited photos, videos, and files

Motive: Designed specifically for file storage

💻
SUSPECT #2: Amazon EC2

Alias: "Elastic Compute Cloud"

Known For: Running applications on virtual servers

Specialty: Provides computing power, not storage

Motive: Could store files, but that's not its main job

🗄️
SUSPECT #3: Amazon RDS

Alias: "Relational Database Service"

Known For: Storing structured data (tables, rows)

Specialty: Databases like MySQL, PostgreSQL

Motive: Stores data, but not files like photos

⚡
SUSPECT #4: AWS Lambda

Alias: "The Serverless Function"

Known For: Running code without servers

Specialty: Executes functions, doesn't store data

Motive: Could process photos, but doesn't store them

📊
SUSPECT #5: CloudWatch

Alias: "The Monitor"

Known For: Monitoring and logging

Specialty: Tracks what's happening in AWS

Motive: Watches everything, but doesn't store files

🎉 CASE SOLVED! 🎉

SOLVED
FINAL REPORT

Culprit: Amazon S3 (Simple Storage Service)

Verdict: INNOCENT - The photos were safe all along!

What Really Happened:

The photos weren't missing! TechCorp simply forgot which S3 bucket they used. Amazon S3 had been faithfully storing all 10,000 photos the entire time.

What You Learned:
  • 📦 Amazon S3 stores files in "buckets"
  • 💻 Amazon EC2 runs applications on virtual servers
  • 🗄️ Amazon RDS manages databases
  • ⚡ AWS Lambda runs code without servers
  • 📊 CloudWatch monitors AWS services
